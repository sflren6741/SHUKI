# AIハンドオフ・テンプレ

> [!note] 📦
>
> **エージェント同士が仕事を引き継ぐときの「引き継ぎ書」テンプレ。**
>
> 会話やチャットは消える。**書いたものだけが残る**。タスクの本文にこのテンプレを貼って埋める。

## 📤 ドラフト提出（専門エージェント → レビュアー）

```
## 📤 ハンドオフ　<作成者エージェント名> → 🔍 レビュアー

**関連タスク**: <タスクページURL>
**ドラフトの目的**: 1文で
**トーン指定**: （例：柔らかく・身を低く・簡潔に）
**参照したコンテキスト**:
- <ページ名とリンク>
- <ページ名とリンク>

---

[ドラフト本文]

---

**迷った点（レビューしてほしいポイント）**:
- <何を迷ったか（1件以上）>

**未解決の不確定事項**:
- <事実関係で検証できなかったもの・オーナーの判断が要るもの>
```

## 🔍 レビュー結果（レビュアー → 作成者 または 人間）

```
## 🔍 レビュー結果 ｜　<ドラフト作成者エージェント名>　へ

**総合判定**: ✅ OK ／ ⚠️ 要修正 ／ ❌ 差し戻し

### 事実レイヤーの指摘
1. <具体的な指摘> → 提案：<代替表現>

### トーンレイヤーの指摘
1. <具体的な指摘> → 提案：<代替表現>

### 価値観レイヤーの指摘
1. <どの価値観と矛盾> → 提案：<ドラフトをどう調整するか>

### アンチAIスロップの指摘
- 「<表現>」はアンチAIスロップリストに該当。代替例：<・・・>

### 良い点、残したいポイント
- <・・・>
```

## 🎯 タスク起票（オーケストレーター → 専門エージェント）

```
## 🎯 タスクディスパッチ　オーケストレーター → <agent名>

**タスクページ**: <リンク>
**背景・状況**: （1～2文）
**completion_criteria**: 1文で明確に
**期限（due）**: <日付>
**ai_judgment_ok**: ✅ true ／ ⛔ false（人間レビュー必要）
**参照すべきコンテキスト**:
- <ページ名とリンク>

**期待する出力フォーマット**: <例：見出し付きGoogle Docsドラフト ／ Notion内ページ ／ チャットに贵り付きテキスト>
```

## 📦 インボックス受け付け（人間 → オーケストレーター）

```
## 📦 インボックス → 🎯 オーケストレーター

<生のメモ・思いつき・取りたいアクション>
```

これを受けたオーケストレーターは**タスクディスパッチテンプレ**に変換して担当エージェントへ振る。

## ⚠️ ハンドオフの原則

- **推測で補わない** — 不確定事項は「未解決」セクションに明記する
- **タスク本文に貼る** — チャットの会話に頂けず、Notionタスクページのコメントまたは本文に記録
- **一タスク一ドラフト** — 複数タスクをひとチャットに混ぜない
- **参照コンテキストをメンションする** — URLはデッドリンクになりやすいので mention-page で
