# 🧠 文脈エンジン構造（PARA運用）

> [!note] 🧠
>
> **全エージェントが共通して読む「会社の脳」。**
>
> どのAIに振っても同じ方針・同じトーン・同じ価値観で出力されるように、判断の元を一箇所に集める。

## 5.1 構造

- 📥 **インボックス**（単一の入口） → [[インボックス]]。タスク・アイデア・資料・文脈・未分類のメモ、すべてここへ一旦集める。オーケストレーターが「種類」を読み、適切な先へ振り分ける
- 🔄 **PARA + アーカイブ**（長期保管）
    - [[Areas]]（継続責任領域。`04_Areas/` 配下の Area 名を列挙）
    - [[Resources]]（蓄積知識）
    - [[アーカイブ]]（完了・引退したもの）
    - 「Projects」はタスクDBの `タイプ=プロジェクト` で受け、重いものは該当Areaのサブページとして作る
- 🎯 **タスクボード** → [[タスク管理]]。オーケストレーターが振り分けた仕事の本体
- 🧭 **コンテキスト3点セット**（イントラの子ページ）：[[哲学・価値観]] / [[デザイン言語]] / [[AIハンドオフ・テンプレ]]

## 5.2 インボックスの自動振り分け方針

オーケストレーターは [[インボックス]] の「種類」を読み、以下へ振り分ける（完了したら `状態 = ✅振り分け済` にし、`振り分け先・推定理由` に送り先URLを残す）：

- 📋 **タスク** → [[タスク管理]] に行を作成（担当エージェント＋完了条件付き）
- 💡 **アイデア** → インボックス内で `状態 = ✅振り分け済` のまま温める（別途動かすときはタスク化）
- 📚 **資料** → [[Resources]]配下のナレッジDBへ送る
- 🧭 **文脈** → このイントラの該当子ページ（哲学・価値観・デザイン言語・AIハンドオフ・テンプレ類）に追記提案を出す
- 🤔 **未分類** → 状態を `❓議論待ち` にしてオーナーに問いを投げる（1件だけ）
