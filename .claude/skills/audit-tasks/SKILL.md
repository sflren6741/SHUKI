# 🧹 タスク棚卸（対話型）

タスクDB（`04_Tasks/タスク管理/タスク/`）を点検し、**検出はAI・判断はオーナー**で一つずつ片付ける。infrastructure エージェントは Read 中心の検出専門だが、このスキルは**会話で決めて、その場で更新まで**行う。

## 🧭 必読コンテキスト
- `00_Intranet/AI運用ルール集（イントラ）/✅ タスク管理ルール.md`（status・担当者・完了条件の定義）
- `CLAUDE.md` §タスクDBの扱い方

## 🔄 フロー

### Step 1: スキャン
`04_Tasks/タスク管理/タスク/*.md` を Glob 全取得し、frontmatter（status / due / priority / created / agent）を抽出して分類：
- 🔴 **期限切れ**: due < 今日 かつ status が未完了
- 🟡 **停滞**: 7日以上更新なし（created/履歴の最終日付基準）かつ未完了
- 🔁 **重複・分裂**: タイトル類似・同テーマの複数ファイル
- 🔗 **参照切れ**: 削除済みエージェント名や存在しないwikilink先

### Step 2: 提示（一覧 → 1件ずつ）
まず件数サマリを出し、次に**1件ずつ**「現状 → 推奨アクション」を提示してオーナーに聞く：
```
🔴 [[タスク名]] — 期限 2026-05-10（21日超過）
  推奨: 延期 / 分解 / 完了扱い / 削除 / 保留 — どれにする？
```
- 1ターンで数件まで。一気に全部出して圧倒させない
- 推奨は出すが、決めるのはオーナー

### Step 3: 決定を反映（オーナーの指示後のみ）
オーナーが決めたものだけ更新する：
- **延期**: `due` を更新、`## 履歴` に1行
- **分解**: 子タスクを新規作成（parent wikilink を張る）、元タスクにリンク
- **完了**: `status: 完了`＋履歴（※オーナーの明示指示がある時だけ。勝手に完了にしない）
- **削除**: オーナーが「削除」と明言した場合のみ。まず内容を見せて確認
- **保留/エスカレーション**: `agent: 人間（オーナー）`＋`ai_judgment_ok: false`

### Step 4: ログ
`00_Intranet/agent-runs/YYYY-MM-DD_HHmm_task-review.md` に処理サマリ（点検N件・更新M件・削除K件）を記録。

## 🚫 やってはいけないこと
- ❌ オーナーの指示なくタスクを削除・完了・統合する（検出と提案まで）
- ❌ 「たぶん停滞」で判定する（必ず日付を確認）
- ❌ センシティブなタスク（個人間コミュニケーション等）の中身に踏み込んだ助言をする（棚卸し対象としての扱いのみ）
- ❌ 一度に大量提示して判断を急かす

## 🔄 セルフQAの3行
```
- 事実チェック: <期限・停滞は日付に基づくか。削除/完了はオーナーの明示指示があったか>
- トーン: <提案ベース・1件ずつ・急かさないか>
- アンチスロップ: <推測停滞・過剰削除提案がないか>
```
