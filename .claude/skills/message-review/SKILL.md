# 💬 メッセージ振り返りレポート（パートナー限定）

前週の**パートナーとの**メッセージ履歴（Facebook / LINE / Instagram のうち今週データがある媒体）を分析し、**まとめレポート**を vault の `09_Logs/ログ/` に作成する。relationship エージェントは Read 専用で書き込めないため、この専用フローがレポート作成を担う。**relationship の作法（事実・パターンのみ／トーン解釈や代筆はしない）を厳守する。**

> 対象はパートナーのみ。前処理 `convert_messages.py` が参加者にパートナー本人を含むスレッドだけを抽出済みなので、CSV にはパートナーとのやり取りしか入っていない。他の人物は対象外。

## 🧭 必読コンテキスト

- `00_Intranet/AI運用ルール集（イントラ）/🪪 Identity.md`
- `00_Intranet/AI運用ルール集（イントラ）/✅ タスク管理ルール.md`（§3.3 代筆禁止ゾーン）
- `CLAUDE.md` §「メッセージ履歴の管理方針」（格納先・命名・person wikilink）

## 📥 入力（前処理済みデータ）

`vault-scripts/convert_messages.py` が前週分を抽出した CSV を読む（**raw 本文 JSON は読まない**）：

| 媒体 | 読むファイル |
|---|---|
| Facebook | `<WORKSPACE>\facebook_messages\data\processed\weekly_latest_facebook.csv` |
| LINE | `<WORKSPACE>\LINE_history\processed\weekly_latest_line.csv` |
| Instagram | `<WORKSPACE>\instagram_messages\data\processed\weekly_latest_instagram.csv` |

各 CSV は `datetime, sender, content`。**今週分が存在する媒体だけ**処理する（ファイルが無ければスキップ）。

## 📤 出力（vault に1ファイル／媒体）

`09_Logs/ログ/<開始日>〜<終了日> <相手名>との<媒体>振り返り.md`
例: `09_Logs/ログ/2026-05-25〜2026-05-31 パートナーとのLINE振り返り.md`

frontmatter（CLAUDE.md 準拠・必須）:
```yaml
---
area: パートナーとの関係        # 相手により調整。パートナー以外は人間関係
date: <開始日 YYYY-MM-DD>  # 期間の開始日
created: <作成日 YYYY-MM-DD>
person: "[[<相手名>]]"     # 例: "[[パートナー]]"
type: 振り返り
媒体: LINE                 # LINE / Facebook / Instagram
---
```

本文構成（事実ベース・簡潔に）:
```markdown
# <期間> <相手名>との<媒体>振り返り

## 概要
- 期間: YYYY-MM-DD 〜 YYYY-MM-DD ／ メッセージ数: N件

## 主な出来事・話題（事実）
- <日付>: <何があったか。固有名詞・予定・決定事項を中心に>

## 約束・予定（事実）
- [ ] <約束/予定>（いつ・何を）

## 関係パターン（観察・仮説）
- <繰り返し見られるやり取りの傾向。「〜かもしれない」と仮説で。トーン採点はしない>

## 気づき（概念ノードへの接続）
- [[<概念ノード>]] — <なぜ繋がるか1行>

## 関連
- [[<相手の人物ノート>]]
```

## 🚫 やってはいけないこと（代筆禁止ゾーンの継承）

- ❌ **パートナー宛の文章・返信案・伝え方の提案を一切書かない**（メッセージ・手紙・添え言葉）
- ❌ トーンの採点・「もっとこう言えば」型の助言
- ❌ パートナーの感情・心理を断定でラベル貼りする（観察は「仮説」に留める）
- ❌ raw の生メッセージ本文を vault にそのまま転記する（要約・事実抽出のみ。長い原文引用をしない）
- ❌ vault 外の raw/processed を編集・削除する（読むだけ。アーカイブ移動は PS1 が行う）
- ❌ お金・健康・外部送信の「アクション」を実行する

## 🔗 wikilink

- `person:` に相手の人物ノートを wikilink（[[パートナー]] 等）
- 「気づき」で関連する概念ノード（`05_Resources/Resources/概念/`）へ接続
- 同じ相手の過去の振り返りログがあれば「関連」で繋ぐ

## 🛑 エスカレーション

CSV が空・破損、相手の特定が曖昧、代筆に踏み込みそうな依頼 → レポートを作らずに orchestrator 経由でオーナーに確認。

```
- アンチスロップ: <代筆に近い表現・心理ラベル・励ましが混入していないか>
```
