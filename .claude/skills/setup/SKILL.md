# 🚀 初回セットアップ（対話型）

このスキルは**ファイルを直接編集しなくてもvaultの初期設定を完了**させる。質問に答えるだけで Identity・Areas・パス設定が整う。

---

## 🧭 フロー概要

```
Phase 1: 現状確認（何が未設定か検出）
Phase 2: 基本プロフィール（Identity.md を完成）
Phase 3: 生活領域（05_Areas/ にハブファイルを作成）
Phase 4: パス設定（CLAUDE.md の <WORKSPACE> 等を置換）
Phase 5: 専門エージェント（任意）
Phase 6: 完了サマリー
```

各フェーズは**1〜2問ずつ**。一気に全項目を聞かない。

---

## Phase 1: 現状確認

まず以下を読んで未設定箇所を洗い出す：
- `00_Intranet/AI運用ルール集（イントラ）/🪪 Identity.md`（`<...>` が残っているか）
- `CLAUDE.md`（`<WORKSPACE>` / `<LOCAL_AI_DIR>` / `<HOME>` が残っているか）
- `05_Areas/` 配下（`_example-area.md` しかなければ Areas 未作成）

検出結果を1行で報告してから Phase 2 に進む：
```
✅ 検出完了：Identity に 7箇所、CLAUDE.md に 3箇所のプレースホルダーが残っています。順番に設定しましょう。
```

すでに設定済みの項目はスキップして未設定分だけ聞く。

---

## Phase 2: 基本プロフィール（Identity.md）

以下を**2問ずつ**聞いていく。全部答えてもらったら Identity.md の完成形を見せて確認を取ってから書き込む。

### 聞く内容（順番に）

1. **名前・呼び方**
   > 「名前（または呼び名）を教えてください。AIにどう呼んでほしいですか？」

2. **言語・タイムゾーン**
   > 「主に使う言語とタイムゾーンは？（例: 日本語、Asia/Tokyo）」

3. **職業・役割**
   > 「職業や役割を簡単に教えてください。AIが文脈を理解するのに使います。細かく書かなくていいです。」

4. **得意技術・ツール**
   > 「AIに前提として知っておいてほしいスキルやツールはありますか？」

5. **ライフステージ・打ち込んでいること**
   > 「今の人生フェーズや、特に打ち込んでいることを教えてください。」

6. **価値観のキーワード**
   > 「行動の軸になっている価値観を2〜3語で教えてください。（例: 誠実さ・成長・自律）」

7. **AIにやらせたくないこと（代筆禁止ゾーン）**
   > 「AIに書かせたくない・判断させたくい領域はありますか？（例: 特定の相手へのメッセージの代筆、金融取引の実行）」

### 書き込み手順

全項目の回答が揃ったら：
1. 完成した Identity.md の本文を**チャットに表示して確認を取る**
2. 「これで保存しますか？」→ OK なら `00_Intranet/AI運用ルール集（イントラ）/🪪 Identity.md` を Edit で書き換える

---

## Phase 3: 生活領域（Areas）

> 「管理したい生活領域を教えてください。（例: 仕事・健康・趣味・英語・家計・人間関係）」

回答例: 「仕事、健康、英語学習」

→ 各 Area につき `05_Areas/<area名>.md` を以下のテンプレートで作成：

```markdown
# <Area名>

## 概要

<Area の目的・スコープを1〜2行>

## フォーカス

- （現在の課題・目標を追記していく）

## ビュー

\`\`\`base
filters:
  area:
    contains: "<Area名>"
\`\`\`
```

作成後、`05_Areas/Areas.md` のハブ（base codeblock）に area 名が反映されることを確認。
既存の `_example-area.md` は残す（削除はオーナー判断）。

---

## Phase 4: パス設定（CLAUDE.md）

自動化（Task Scheduler 等）を使う予定があるかどうかを確認：

> 「朝のスタンドアップや夜間QAを自動実行する予定はありますか？（任意・あとでも設定できます）」

**使う予定あり**の場合：
- `<WORKSPACE>` — 作業用フォルダのパス（例: `C:\Users\username\workspace\VSCode`）
- `<LOCAL_AI_DIR>` — Ollama スクリプトのフォルダ（使わない場合は削除）
- `<HOME>` — ホームディレクトリ（例: `C:\Users\username`）

パスを聞いて CLAUDE.md の該当箇所を Edit で置換。

**使う予定なし**の場合：
プレースホルダーをコメント付きで残す or 該当行ごと削除するかオーナーに確認。

---

## Phase 5: 専門エージェント（任意）

> 「コア4体エージェントに加えて、専門エージェントを追加しますか？例えば：家計管理・語学学習・計画・内省など。（スキップも可）」

追加する場合は、役割を聞いて `.claude/agents/<name>.md` を `📋 エージェント共通テンプレート.md` に沿って作成する。

---

## Phase 6: 完了サマリー

設定した内容を箇条書きで報告：

```
✅ セットアップ完了

設定したもの：
- Identity.md（プロフィール・価値観・境界線）
- Areas: 仕事 / 健康 / 英語学習（各ハブファイル作成済み）
- CLAUDE.md パス設定（<WORKSPACE> 置換済み）
- 専門エージェント: household（家計管理）

次のステップ（任意）：
- Obsidian プラグイン設定 → docs/OBSIDIAN-SETUP.md
- 自動化（Task Scheduler）→ docs/AUTOMATION.md
- 日常利用を始める → README の「日常の使い方」
```

---

## 🚫 やってはいけないこと

- ❌ 確認なくファイルを書き換える（必ず完成形を見せてから書き込む）
- ❌ 一度に全項目を聞く（2問ずつ・圧倒させない）
- ❌ Identity.md に書くべきでない個人情報（パスワード・連絡先等）を記入させる
- ❌ 代筆禁止ゾーンに関わる内容を自動で記入する

## 🔄 セルフQAの3行

```
- 事実チェック: <プレースホルダーが全て置換されたか。書き込み前に確認を取ったか>
- トーン: <親しみやすく、圧倒させていないか>
- アンチスロップ: <聞いていない情報を勝手に補完していないか>
```
