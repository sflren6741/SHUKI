# 📥 インボックス

> 何でもここに放り込む。`@orchestrator` が定期的に振り分け・frontmatter補完を行い、正式な置き場所（タスク・ログ・ナレッジ・プロジェクト）へ転記する。  
> `status: done` にするのは情報の転記が完了してから。判断軸は「削除するか？」ではなく「内容が移行済みか？」。

```base
filters:
  and:
    - file.inFolder("01_Inbox/インボックス")
views:
  - type: table
    name: 📥 未処理
    filters:
      and:
        - status.containsAny("new", "routing")
    groupBy:
      property: status
      direction: ASC
    order:
      - file.name
      - status
      - type
      - area
      - created
    sort:
      - property: created
        direction: DESC
  - type: table
    name: ⏸ 保留
    filters:
      and:
        - status.containsAny("on-hold", "pending")
    order:
      - file.name
      - type
      - area
      - created
    sort:
      - property: created
        direction: ASC
  - type: table
    name: 📋 全件
    groupBy:
      property: status
      direction: ASC
    order:
      - file.name
      - status
      - type
      - area
      - created
    sort:
      - property: created
        direction: DESC
```
