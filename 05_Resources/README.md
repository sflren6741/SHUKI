# 05_Resources — Reusable knowledge & concept nodes / ナレッジ＋概念ノード

**EN**
- `Resources/ナレッジ/` — reusable knowledge entries, one file per entry, classified by `type:` (📕book / 📰article / 📜principle / 📺anime / 📖manga / 🎮game / 🎵music / 📺movie / **👤person**). Yes — **people are knowledge notes too** (`type: 👤人物`): one file per person, holding the facts you want to remember about them. Logs link to them via `person: "[[<name>]]"`, so Obsidian's backlinks panel aggregates everything about that person automatically.
- `Resources/概念/` — **concept nodes**: tiny notes for themes that recur across the vault. These become the semantic hubs of the graph; create one when a theme shows up in 3+ files.

**JP**
- `Resources/ナレッジ/` — 再利用可能なナレッジ。1ファイル1エントリ、`type:` で分類（📕本 / 📰記事 / 📜原則・ルール / 🎮ゲーム … / **👤人物**）。**人物も「ナレッジノート」として管理する**（`type: 👤人物`）：1ファイル1人物に、その人について覚えておきたい事実を集約。ログ側で `person: "[[<名前>]]"` と書くと backlinks で自動集約される。
- `Resources/概念/` — **概念ノード**：vault 横断で繰り返し登場するテーマの小さなノート。グラフの意味的ハブになる（同テーマが3ファイル以上に出たら作る）。

> Skeletons: `ナレッジ/_example-person.md`（人物ノート） / `概念/_example-concept.md`（概念ノード）。
